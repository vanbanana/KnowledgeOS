# 产品需求文档 PRD

## 一、产品基本信息

**产品暂定名**：KnowledgeOS / 学习知识工作台  
**文档版本**：V1.0  
**产品阶段**：立项 / MVP 定义  
**目标用户**：学生、研究生、科研人员、课程学习者、知识工作者  
**产品定位**：  
一个面向学习与研究场景的本地优先 AI 知识工作台。  
它将用户导入的课程资料、论文、讲义、PPT、Word、Markdown 等统一转为 Markdown 和块级知识单元，通过 AI 辅助阅读、自动摘要、知识图谱构建、本地文件 Agent 整理和结构化复习产物生成，帮助用户把“资料”转化为“可积累、可关联、可复用的知识网络”。

---

## 二、产品愿景

用户今天最大的痛点不是“没有资料”，而是：

- 资料很多，但阅读是割裂的
    
- AI 能解释文档，但解释无法沉淀
    
- 笔记能记录内容，但很难自动组织成结构
    
- 知识图谱能看关系，但不能真正辅助深度阅读
    
- 本地文件越来越乱，整理成本极高
    

本产品的目标不是做一个普通的 AI 文档对话工具，而是做一个：

**把资料导入、阅读理解、知识连接、文件整理、复习输出串成闭环的学习操作系统。**

---

## 三、产品目标

### 3.1 核心目标

在一个统一工作台内，完成以下闭环：

**导入资料 → 统一转换 → 分块阅读 → 块级 AI 解读 → 生成卡片/笔记 → 构建知识网络 → 自然语言整理知识库 → 输出学习成果**

### 3.2 业务目标

MVP 阶段重点验证三件事：

1. 用户是否愿意把课程资料/论文/讲义集中导入本产品
    
2. 用户是否认可“分块阅读 + 块旁 AI 解读”的体验
    
3. 用户是否会持续使用“知识图谱 + Agent 整理”的能力来沉淀长期知识库
    

### 3.3 非目标

MVP 阶段暂不做：

- 通用社交社区
    
- 在线多人协作编辑
    
- 复杂项目管理功能
    
- 通用办公套件替代
    
- 全量跨设备云同步
    
- 面向企业的大规模权限系统

## 四、目标用户与典型场景

## 4.1 用户画像

### 用户 A：课程学习型学生

需求：导入老师 PPT、讲义 PDF、课本章节，快速理解重点，生成复习笔记和知识结构。  
痛点：资料散、重点难抓、AI 聊天内容看完就没了。

### 用户 B：科研型研究生 / 研究者

需求：导入论文、代码说明、实验文档，按概念和研究主题建立知识网络。  
痛点：论文阅读负担大，前后文关联困难，论文与代码理解断裂。

### 用户 C：自学型知识工作者

需求：导入视频讲义文稿、电子书、笔记文章，构建自己的长期知识库。  
痛点：看过很多内容，但无法真正积累成体系。

---

## 4.2 核心使用场景

### 场景 1：课程资料深度学习

用户导入一门课的 PPT、PDF 讲义、参考文档。  
系统将其统一转为 Markdown 并分块。  
用户在阅读每个知识块时，右侧看到 AI 的块级解释、重点、术语说明、先修知识和例题。  
用户可把解释一键保存为知识卡片，并自动进入图谱。

### 场景 2：论文精读与知识沉淀

用户导入一篇论文和相关补充材料。  
系统将论文拆成摘要、背景、方法、实验、结论等块。  
用户可逐块阅读，每块旁边都有 AI 解释，并能把某一块与其他论文/课程内容建立关联。  
最终形成“论文知识网络”而非一次性对话记录。

### 场景 3：自然语言整理知识库

用户输入：“帮我把这门课第 1-3 章重复概念合并，并按照先修关系重排。”  
本地 Agent 在授权下完成文件重命名、卡片合并建议、知识链接补全和目录优化。  
用户确认后写回本地知识库。

## 五、产品核心价值

本产品要同时覆盖四类价值：

### 5.1 学习价值

帮助用户从“读资料”走向“真正理解并形成结构”。

### 5.2 知识价值

将一次性的 AI 对话，沉淀成长期可复用的知识节点和关系网络。

### 5.3 效率价值

通过本地 Agent 和自然语言指令，降低整理文件、重组笔记、生成复习材料的成本。

### 5.4 隐私价值

用户资料默认保存在本地，知识库可自主管理，不依赖纯云端平台。

## 六、产品差异化定位

本产品不是单一功能的替代品，而是融合三种范式：

### 6.1 相比 NotebookLM 类产品

差异点在于：  
不仅做基于资料的问答和学习产物生成，还要把资料长期结构化沉淀到本地知识库，并形成图谱与可编辑节点。

### 6.2 相比 Obsidian 类产品

差异点在于：  
不仅是 Markdown 笔记工具和图谱工具，而是从“原始资料导入”开始自动做分块、解释、知识提取和结构构建。

### 6.3 相比 Vibero 类产品

差异点在于：  
不仅强调空间化阅读和深度理解，还要把阅读过程中的理解结果沉淀为长期可检索、可整理、可复习的知识资产。

### 6.4 相比本地代码 Agent

差异点在于：  
不是面向代码仓库，而是面向知识文件、课程资料、学习笔记和概念关系的本地知识 Agent。

## 七、产品原则

1. **本地优先**：用户文件和知识库默认存于本地
    
2. **源文件可追溯**：所有 AI 解释都可回溯到原文块
    
3. **块级理解优先**：不鼓励直接全篇泛化总结，优先逐块深读
    
4. **知识沉淀优先**：AI 输出默认可保存为卡片、链接、图谱节点
    
5. **可控自动化**：Agent 对本地文件的修改必须可预览、可确认、可回滚
    
6. **兼容开放生态**：优先兼容 Markdown、JSON Canvas、常见本地目录结构

## 八、功能范围

## 8.1 MVP 功能范围（必须有）

### 模块 A：资料导入与统一转换

支持导入：

- PDF
    
- PPT / PPTX
    
- DOC / DOCX
    
- Markdown
    
- TXT
    

系统能力：

- 将资料统一转换为 Markdown 中间格式
    
- 保留标题层级、段落、页码/页序号、图片位置、表格结构引用
    
- 为每个文档生成唯一文档 ID
    
- 为每个块生成唯一块 ID
    
- 保留源文档与 Markdown 的映射关系
    

### 模块 B：分块阅读器

系统将导入内容切分为多个知识块。  
每个知识块可独立阅读、评论、解释和保存。

块的切分维度包括：

- 标题层级
    
- 段落语义
    
- 幻灯片页
    
- 图片/图表说明
    
- 表格段
    
- 公式区域（V1 可先弱支持）
    

阅读界面要求：

- 左侧为原块内容
    
- 右侧为 AI 解读面板
    
- 支持上下块导航
    
- 支持“回到原文位置”
    
- 支持块收藏、标注、加入知识库
    

### 模块 C：块级 AI 解读

每个块旁边提供 AI 解读，不是对整篇文件一次性总结。

块级解读默认包含：

- 本块摘要
    
- 关键概念
    
- 术语解释
    
- 这段内容在整份资料中的作用
    
- 先修知识提醒
    
- 易错点 / 难点提示
    
- 与已有知识库中内容的关联建议
    

可扩展解读方式：

- 用更简单的话解释
    
- 面向考试视角解释
    
- 面向科研视角解释
    
- 给例子
    
- 生成提问
    
- 生成记忆卡片
    

### 模块 D：知识卡片与知识图谱

用户可将任意块、任意 AI 解读内容保存为知识卡片。  
卡片进入知识库后，可建立关系：

- 定义
    
- 属于
    
- 前置知识
    
- 因果
    
- 对比
    
- 举例
    
- 延伸
    
- 来源于同一课程/论文/主题
    

图谱要求：

- 支持按课程、主题、文档过滤
    
- 支持搜索节点
    
- 支持点击节点回看来源块和 AI 解读
    
- 支持自动推荐关联边
    
- 支持手动编辑关系
    

### 模块 E：本地知识 Agent

用户可通过自然语言操作知识库和本地资料目录。

MVP 支持的 Agent 指令类型：

- 帮我整理这门课的资料目录
    
- 按章节重命名这些笔记
    
- 把重复概念合并成一张主卡片
    
- 为这些块补充标签
    
- 根据内容自动生成课程知识树
    
- 生成本周复习清单
    
- 把这个项目导出成 Markdown 知识库结构
    

Agent 执行要求：

- 所有改动前先给预览
    
- 需要用户确认后写入
    
- 支持撤销和变更日志
    

### 模块 F：项目级知识库

每个学习主题或课程作为一个 Project。  
Project 内包含：

- 原始资料
    
- 转换后的 Markdown
    
- 块列表
    
- 知识卡片
    
- 图谱关系
    
- AI 解读缓存
    
- 用户标注与历史操作

## 九、典型用户流程

## 9.1 新用户首个使用流程

1. 创建项目
    
2. 导入课程资料文件夹
    
3. 系统自动解析并转换为 Markdown
    
4. 系统自动分块并生成初始目录结构
    
5. 用户进入阅读器逐块阅读
    
6. 用户保存重点块为知识卡片
    
7. 系统自动生成初始知识图谱
    
8. 用户通过 Agent 指令继续整理和扩展知识库
    

## 9.2 深度阅读流程

1. 打开某个知识块
    
2. 查看右侧 AI 解读
    
3. 切换“考试解释”或“研究解释”
    
4. 对该块进行标注或提问
    
5. 选择“保存为卡片”
    
6. 与已有概念建立关联
    
7. 在图谱中查看该知识点的上下游关系
    

## 9.3 知识整理流程

1. 用户输入自然语言任务
    
2. Agent 生成整理方案
    
3. 展示受影响文件/节点/关系
    
4. 用户确认
    
5. 系统执行
    
6. 生成变更记录
    
7. 用户可撤销

## 12.2 验收标准

### FR-01 文件导入

- 用户可导入至少 5 种主流学习资料格式
    
- 导入成功率在常见规范文件上达到可用水平
    
- 用户可查看导入状态和失败原因
    

### FR-03 块切分

- 系统能按标题、段落和页面切分
    
- 每个块有明确边界
    
- 用户能看到块与原文位置映射关系
    

### FR-04 块级 AI 解读

- 每个块都能生成结构化解释
    
- 解释必须绑定来源块
    
- 用户可再次追问该块，而非整篇泛问
    

### FR-07 本地 Agent

- Agent 在执行前必须展示拟执行内容
    
- 涉及文件改动时必须二次确认
    
- 用户可查看修改日志并撤销

## 十三、技术与系统要求

## 13.1 推荐架构方向

### 客户端

桌面端优先，建议采用：

- Electron 或 Tauri
    
- 前端框架建议 React / Next 风格组件体系

建议包含以下本地服务层：

- 文件监听器
    
- 文档解析器
    
- Markdown 正常化引擎
    
- 块切分引擎
    
- 向量索引服务
    
- 图谱关系引擎
    
- Agent 调度器
    
- 模型适配层
    

### 数据存储

本地数据库建议：

- SQLite：项目、块、卡片、关系、日志
    
- 本地文件系统：原文与 Markdown 中间件
    
- 向量库：用于语义检索
    
- 图结构表：用于知识关系管理

- 中小型课程资料导入应可接受

- 用户文件默认本地存储

- 提供项目级数据导出和删除能力
    

### 可维护性

- 文档转换层与 AI 层解耦
    
- Agent 执行器与 UI 解耦
    
- 数据结构保持开放，便于兼容 Markdown / Canvas / Obsidian 生态

## 十五、核心交互原则

1. 所有 AI 输出必须尽量“贴着原文块”发生
    
2. 所有知识沉淀必须允许“一键写回知识库”
    
3. 所有自动整理动作必须“先预览后执行”
    
4. 用户永远知道某条解释、某张卡片、某条关系来自哪里
    
5. 图谱不是装饰层，而是阅读和复习的导航层
    

---

## 十六、MVP 版本定义

## 16.1 MVP 必须交付的最小闭环

用户可以完成以下整套流程：

1. 导入一门课的资料
    
2. 系统转成 Markdown 并切块
    
3. 用户逐块阅读并看到 AI 解读
    
4. 用户将重点保存为知识卡片
    
5. 系统自动生成初始图谱
    
6. 用户通过 Agent 整理项目结构
    
7. 用户导出整理后的本地知识库
    

### MVP 的核心判断标准

只看三件事：

- 用户能不能比普通 PDF 阅读更快进入理解状态
    
- 用户愿不愿意把 AI 解释保存成长期知识资产
    
- 用户会不会反复回来维护这套知识网络

## 十九、风险与应对

### 风险 1：文档转换质量不稳定

应对：  
先聚焦常见教学资料格式，建立 Markdown 中间层和人工校验机制。

### 风险 2：图谱看起来很炫但没有实际价值

应对：  
图谱必须和阅读器、卡片、搜索、复习流程强绑定，不做纯展示型图谱。

### 风险 3：Agent 改坏本地文件

应对：  
必须有预览、确认、日志、回滚四层保护。

### 风险 4：AI 解读流于空泛

应对：  
优先做块级解释模板化，严格绑定来源块，不先追求“全能聊天”。

### 风险 5：功能过重导致首版过大

应对：  
首版只做一条主链路，不同时做社区、协作、移动端、复杂同步。

## 二十、结论

这个产品最核心的创新点，不是“又一个 AI 阅读器”，而是把以下四件事真正做成一个统一系统：

**资料导入标准化**  
**分块深度阅读**  
**知识图谱沉淀**  
**本地 Agent 整理执行**

一句话概括这份 PRD：

**这是一个面向学习与研究场景的本地优先 AI 知识工作台，让用户把课程资料和论文资料从“文档”变成“知识网络”。**

下一步最适合先把这份 PRD 继续拆成 **MVP 功能清单 + 页面结构图 + 技术架构图**，这样就可以直接进入设计和研发排期。
